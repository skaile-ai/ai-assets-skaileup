Findings from the 2026-05-10 catalog review across all 17 domains, 94 SKILL.md
files, the contracts/, flows/, bundles/ layers, and the top-level reference
docs. Items are grouped by **severity × cost-to-fix**.

## A. Critical — block other improvements

| ID | Issue | Action |
|---|---|---|
| **A1** | `CLAUDE.md` (path-prefix names) and `CONTRIBUTING.md` (parent-dir match) contradict each other | Rewrite `CONTRIBUTING.md` to match the path-prefix convention the implementation uses |
| **A2** | 19 skills missing `metadata.version` | Bulk-add `1.0.0` (or `0.1.0` for alpha) |
| **A3** | 51 skills missing `metadata.stage` | Bulk-add `alpha` / `beta` based on validator + tests presence |
| **A4** | 18 skills still use deprecated `metadata.user_inputs` | Mechanical migration to `prerequisites.inputs_required` / `inputs_optional` |
| **A5** | Templates classified as skills but contain only reference content (420–720 lines, no procedural steps) | Convert to a new `template:` asset kind, or to plain `.md` references |

## B. High — worth doing in the next pass

| ID | Issue |
|---|---|
| **B1** | concept-slice/ skills are alpha stubs with truncated workflows (brainstorm stops at STEP 4, align at STEP 6) |
| **B2** | mockup-feedback loop is one-way: mockup-walkthrough-* regenerators don't read `_feedback/devlog.md`, so regenerations lose human intent |

| **B4** | `_grounding/general/` paths used in 4 concept-side skills; new research skill writes to `_grounding/research/` (path mismatch) |

## C. Medium — improves coherence

- **C1** — All 20 DOMAIN.md files are stubs (Phase 2 deliverable per SKILL_GRAPH.md). Author with: when the domain runs, inputs/outputs, dependencies, example call.
- **C2** — Description quality: ~30% follow "Use when..." pattern; the rest are statement-style. Hurts the agent's skill router.
- **C3** — DSL grammar adoption is uneven: 57/94 use ROLE, 49/94 use CHECKLIST, 60/94 use MUST. Run `lab/judge` against `contracts/skill_grammar.md` to score and backfill.
- **C4** — `eval-{concept,feature,product}` should expose their gate position via `metadata.gate`.
- **C5** — `mockup-feedback-patch` references `mockup-feedback/schemas/*.schema.json` files that don't exist in the repo.
- **C6** — Document the "narrowly scoped validator" pattern (`lab/validate-elements-block`) explicitly in `lab/DOMAIN.md`.

## D. Low — cosmetic

- **D1** — `contracts/MIGRATION.md` line 6 references the Phase-0 path `skaileup-contracts/scripts/...` (now `contracts/scripts/...`).
- **D2** — README says "17 domains" but counting sub-domains the answer is closer to 20–22. Pick a definition.
- **D3** — Description length range is too wide (8 chars for DOMAIN.md to ~190 chars for some skills). Add a CI lint.
- **D4** — README + CLAUDE.md + SKILL_GRAPH.md mention `mockup-walkthrough-lit` but the skill doesn't exist.
- **D5** — `validator.py` coverage is 38/94 with no enforced rule — should be required for `stage: stable`.

## E. Splits / merges / removals to consider

| ID | Verdict | Rationale |
|---|---|---|
| `impl-quality/standards-*` (3 skills) | **keep split** | Each phase is individually invokable from flows |
| `impl-slice/{commit, git-prepare, finish}` | **keep split** | Different lifetimes (per-slice vs. project bookends) |
| `experience/screens-technical` | **remove or archive** | Marked experimental, no flow uses it |
| Templates as skills | **convert to non-skill assets** | 400–700-line reference docs, no STEPs |
| `mockup-walkthrough-text` | **consider splitting** | One file covers Alpine + Vue + Preact + Shoelace |
| `mockup-walkthrough-{lit, framework}` | **add** | Referenced in tier tables, files don't exist |
| `impl-slice/implement/impl-slice-implement-page` | **promote to sibling** | Only nested skill in the catalog; structurally inconsistent |

## F. Process improvements

- **F1** — CI gate: frontmatter audit (fail on missing version/stage/tags, deprecated fields, stable-without-validator).
- **F2** — CI gate: bundle ↔ flow drift via existing `scripts/check-bundles.sh`.
- **F3** — Pre-commit hook: `grep -rE 'skaileup-|_grounding/general/' --include='SKILL.md'` to catch path drift after renames.
- **F4** — Rename `REFACTOR_MOCKUP.md` → `docs/mockup-design.md` (the refactor landed).
