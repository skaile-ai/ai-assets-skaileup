The catalog reached its current shape through four reorganization phases. All
are complete; this page is the historical record.

<Aside type="tip">
The flow verifier (`flows/_meta/verify_flows.py`) and the frontmatter audit
(`docs/scripts/audit.py`) are the live source of truth for catalog health.
`deferred_skills.yaml` is empty — there is no outstanding tier-flow work.
</Aside>

## Phase 2 — quality remediation (complete)

The 2026-05-10 catalog review backlog was worked through across five commits:

- **Frontmatter** — `version` + `stage` completed on every skill; deprecated
  `user_inputs` → `prerequisites` migrated; deprecated root
  `reads_from`/`writes_to` removed.
- **Templates demoted** — the 7 stack templates are reference assets
  (`TEMPLATE.md`, `type: template`), no longer skills; resolved at runtime by
  [`impl-architecture-templates-select`](../domains/impl-architecture/impl-architecture-templates-select/).
- **DOMAIN.md** — authored for every domain (no more stubs).
- **Descriptions** — normalized toward the "Use when…" router pattern.
- **Naming** — `CLAUDE.md` and `CONTRIBUTING.md` reconciled on the path-prefix
  convention.
- **CI gates** — frontmatter audit, bundle↔flow drift guard
  (`check-bundles.sh`), and a pre-commit hook.

## Phase 3 — SKILL_GRAPH migration (complete)

All 11 deferred tier-flow skills built; `deferred_skills.yaml` is now empty and
the flow verifier emits no deferred-skill warnings:

- [`concept-goals`](../domains/concept/concept-goals/),
  [`concept-comparable`](../domains/concept/concept-comparable/),
  [`design-inspiration`](../domains/design/design-inspiration/) — high-level
  concept pass for standard/complex tiers.
- [`impl-architecture-templates-select`](../domains/impl-architecture/impl-architecture-templates-select/)
  — runtime selector over the 7 `template-*` reference assets.
- [`mockup-walkthrough-astro`](../domains/mockup-walkthrough/mockup-walkthrough-astro/) ·
  [`-lit`](../domains/mockup-walkthrough/mockup-walkthrough-lit/) ·
  [`-framework`](../domains/mockup-walkthrough/mockup-walkthrough-framework/) and
  the `mockup-feedback` cluster (annotate · triage · patch · apply) — walkthrough
  renderers + feedback loop.

## Known follow-ups

Small, non-blocking items tracked outside the phase plan:

- **Tier-shape extras** — `simple/standard/complex` bundles inherit
  `mockup-walkthrough-text` (and `-isolated-html`) that those tiers don't run in
  their flow node graph. Installing the inherited variant is a no-op; the
  verifier reports it as a warning by design.
- **Validator coverage** — roughly half the skills ship a `validator.py`; no
  rule yet requires one for a given stage.
