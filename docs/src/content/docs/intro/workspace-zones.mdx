The slice loops only work if scratch handoffs don't pollute permanent state.
The collection defines three filesystem zones with distinct lifetimes.

```
   ┌─────────────────────────┬───────────┬──────────────────────────────┐
   │ Zone                    │ Lifetime  │ Contents                     │
   ├─────────────────────────┼───────────┼──────────────────────────────┤
   │ _concept/               │ permanent │ Product spec — features,     │
   │                         │           │ screens, datamodel, brand    │
   │ _concept/_meta/         │ permanent │ scope.yaml (chosen tier)     │
   │ _implementation/        │ permanent │ PLANS.md, progress.json,     │
   │                         │           │ decisions.md                 │
   ├─────────────────────────┼───────────┼──────────────────────────────┤
   │ _concept/slices/<id>/   │ per-slice │ concept-slice dossier:       │
   │                         │ (frozen,  │   brainstorm.md align.md     │
   │                         │  kept)    │   scope-feature.md index.md  │
   │                         │           │ ► frozen by concept-slice/   │
   │                         │           │   design-feature (index.md   │
   │                         │           │   written; folder kept)      │
   ├─────────────────────────┼───────────┼──────────────────────────────┤
   │ _implementation/        │ per-slice │ impl-slice dossier:          │
   │   slices/<id>/          │ (frozen,  │   brainstorm.md align.md     │
   │                         │  kept)    │   plan.md test.md recap.md   │
   │                         │           │   refactor.md index.md       │
   │                         │           │ ► frozen by impl-slice/      │
   │                         │           │   commit (only progress.json │
   │                         │           │   removed; folder kept)      │
   ├─────────────────────────┼───────────┼──────────────────────────────┤
   │ _feedback/              │ permanent │ mockup-feedback loop state — │
   │   sessions/<sid>.json   │ gitignore │ raw annotations              │
   │   patches/<sid>.json    │ gitignore │ proposed diffs               │
   │   applied/<sid>.json    │ committed │ audit trail of applied       │
   │   devlog.md             │ committed │ append-only narrative log    │
   │                         │           │ (agent reads on regenerate,  │
   │                         │           │  filtered by mentioned-path) │
   │   devlog.archive/       │ committed │ rollups every 500 entries    │
   └─────────────────────────┴───────────┴──────────────────────────────┘
```

## Lifetimes

- **Permanent** zones survive every reset. They are the source of truth the
  agent reads from after a `/clear`.
- **Per-slice** dossiers are **frozen** (not deleted) by the closing skill of
  their loop (`design-feature` for concept-slice, `commit` for impl-slice): the
  closing skill writes an `index.md` and keeps the phase handoffs as permanent
  per-feature documentation. A dossier with an `index.md` is frozen; one without
  is still in flight. (Only the impl side's transient `progress.json` is removed.)
- **Permanent + gitignored** (`_feedback/sessions/`, `_feedback/patches/`)
  hold transient session state that doesn't belong in version control but
  still needs to outlive a `/clear`.

## Why _concept/ is single-source

Every concept artifact has exactly one home in `_concept/`:

```
_concept/
├── _meta/
│   └── scope.yaml                       ← chosen tier, set by scope-project
├── discovery/
│   ├── brief.md                         ← elevator pitch, audience
│   ├── goals.md
│   └── comparable.md
├── design/
│   ├── identity.md
│   ├── tokens.json                      ← brand tokens
│   └── voice.md
├── experience/
│   ├── features/<group>/<feature>.md
│   ├── screens/<group>/<screen>.md
│   ├── journeys/stories.json
│   └── components/...
├── mockup-walkthrough/
│   ├── text/                            ← ASCII wireframes
│   ├── static-html/                     ← zero-build HTML
│   └── astro/                           ← Astro project (default for appbuilder-standard)
├── mockup-component/
│   ├── isolated-html/
│   └── storybook/
├── slices/<id>/                         ← per-feature concept dossier (frozen, kept)
│   ├── brainstorm.md align.md scope-feature.md
│   └── index.md                         ← links to the canonical feature/screens/walkthrough
└── _feedback/                           ← see above
```

The screen-card pattern keeps mockup variants traceable: a single screen
spec at `experience/screens/login.md` has corresponding files at
`mockup-walkthrough/text/login.txt`, `static-html/login.html`,
`astro/login.astro`, and `mockup-component/storybook/Login.stories.ts`.

**Shared filename stem ⇒ traceability is mechanical.** Rendered DOM carries
`data-spec-screen` and `data-spec-element` attributes that the
mockup-feedback loop uses to resolve annotations back to the right artifact.
