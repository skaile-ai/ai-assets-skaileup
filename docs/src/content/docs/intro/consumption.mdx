The collection is consumed by the [`skaile` CLI](https://github.com/skaile-ai/skaile-agent-framework)
via `@skaile/asset-manager`, which scans a repository by walking every file
and building a catalog from recognized manifest files.

## Manifest file types

| Filename | Asset kind |
|---|---|
| `SKILL.md` | `skill` |
| `AGENT.md` | `agent` |
| `agent.yaml` | `agent` |
| `CONTRACT.md` | `contract` |
| `MCP.md` | `mcp-server` |
| `CONNECTOR.md` | `connector` |
| `MOUNT.md` | `mount` |
| `<name>.prompt.md` | `prompt` |
| `<name>.bundle.yaml` | `bundle` |
| `<name>.flow.yaml` | `flow` |

Each manifest file produces exactly one catalog entry. Name collisions (same
`kind:name`) are resolved by picking the alphabetically later source path.

## Deploy targets

Project-local install paths per agent driver:

| Driver | Skill | Agent | Prompt | Contract | Flow |
|---|---|---|---|---|---|
| `claude-code` | `.claude/skills/` | `.claude/agents/` | `.claude/commands/` | `.claude/contracts/` | `.skaile/flows/` |
| `omp` | `.omp/skills/` | `.omp/agents/` | `.omp/prompts/` | `.omp/contracts/` | `.skaile/flows/` |
| `codex` | `.codex/skills/` | `.codex/agents/` | `.codex/rules/` | `.codex/contracts/` | `.skaile/flows/` |

Local repos (declared with `path:` in `skaile.yaml`) are **symlinked**;
remote repos (declared with `url:`) are **copied**. Symlinks update live;
copies only update when content changes.

## Registering this repo in a workspace

In the project's `skaile.yaml`:

```yaml
repositories:
  # Remote (install by copying)
  ai-assets-skaileup:
    url: https://github.com/skaile-ai/ai-assets-skaileup
    branch: main

  # Local checkout (install by symlink — live updates)
  ai-assets-skaileup:
    path: ../ai-assets-skaileup
```

Then install assets — everything, or just one flow's dependencies (each flow
carries its own `requires:` manifest):

```bash
$ skaile add skill:*                    # install every skill (whole collection)
$ skaile add flow:appbuilder-simple            # OR install exactly what appbuilder-simple needs
$ skaile add skill:concept-brief        # OR install one skill + its requires
$ skaile add --global skill:my-skill    # install globally (~/.claude/skills/)
$ skaile list                           # show installed assets
$ skaile search skill concept           # search the catalog
```

## Running flows

Installing skills doesn't run anything. Once installed, a flow (`<name>.flow.yaml`)
runs **two interchangeable ways**:

```bash
$ skaile run flow:appbuilder-simple            # 1. the skaile workspace flow engine (connector)
```

2. **the orchestrator** (`skaileup` / `skaileup-build`) — it understands the flow
   files and guides/executes the same node graph conversationally. Use it when no
   flow engine is present, or when you want a human-in-the-loop run.

See [Flows](/intro/flows-and-bundles/) for the node-graph details.

## Submodule layout in skaile-dev

The shell repo `skaile-dev-matthias/` mounts the collection as a git submodule:

```
skaile-dev-matthias/
├── ai-assets/                 ← general-purpose skills
└── ai-assets-skaileup/        ← THIS REPO — Skaile-specific collection
```

Both are mounted in workspace flows alongside the agent framework runtime
(`skaile-agent-framework/`).
