The **Implementation group** is the half of the catalog that *builds* the
product. It has a one-time project setup and a repeating per-feature
[`impl-slice`](../../flows/impl-slice/) loop — the same rhythm as the concept
side.

## Domains

- [**impl-architecture**](./impl-architecture/) — techstack · system · datamodel · templates
- [**impl-plan**](./impl-plan/) — brainstorm · align · plan-vertical · supervised
- [**impl-build**](./impl-build/) — one-time setup: scaffold · foundation · infrastructure · migrate · seed · generate · docs
- [**impl-slice**](./impl-slice/) — the per-feature loop: implement · test · recap · refactor · commit
- [**impl-quality**](./impl-quality/) — test-* · eval-code · audit · ready · standards-* · debug-*
